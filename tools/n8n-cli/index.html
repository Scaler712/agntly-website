n8n-cli

AI-agent-friendly CLI for n8n workflow automation. Node-level inspect, patch, and update operations via a parser abstraction layer — so agents can modify individual nodes without rewriting entire workflows.

View on GitHub How It Works
The Problem

n8n's API was never built
for agents

The native n8n REST API only offers workflow-level CRUD. To change a single node, you have to GET the full workflow JSON, find the node, edit it, and PUT the entire blob back. That's fine for humans — but completely impractical for AI agents that need to make surgical, targeted changes.

# Without n8n-cli: the painful way
1. GET /api/v1/workflows/123
2. Parse 2,000+ lines of JSON
3. Find "HTTP Request" node in nodes[]
4. Modify the URL parameter
5. PUT entire workflow JSON back
 
# Hope nothing else changed in between
# Hope you didn't break a connection
# Hope the JSON is still valid
# With n8n-cli: one command
$ n8n-cli node update 123 n1 --set parameters.url=https://new-api.com
 
Patching node n1 "HTTP Request"...
parameters.url:
- https://old-api.com/v1
+ https://new-api.com
 
✓ Node updated successfully
The Solution

A parser layer that gives agents
node-level control

n8n-cli builds an abstraction on top of the standard n8n API. It parses workflow JSON, assigns stable references to every node, lets agents write targeted patches for individual nodes, and handles the full workflow update under the hood.

Capabilities

What agents can do
with n8n-cli

Inspect Nodes

List all nodes, get structured details, extract specific parameter values by dot-path. Multiple output views: summary, details, JSON, params, connections.

n8n-cli node get <wf> n0 --view params

Surgical Patches

Update individual parameters with --set, remove with --unset, or deep-merge from a JSON file. Each update shows a before/after diff of changed fields.

n8n-cli node update <wf> n1 --set parameters.url=...

Stable References

Every node gets a stable ref (n0, n1, ...) by array position. Agents can address nodes without fragile name lookups. Also supports id: and name: selectors.

ref:n0 | id:abc-123 | name:"HTTP Request"

Graph Analysis

Analyze workflow topology: roots, leaves, topological order, cycle detection, branching points, and orphan nodes. Agents can reason about workflow structure.

n8n-cli graph inspect <wf> --with-cycles

Create & Connect

Create nodes from JSON or CLI flags, wire connections with specific input/output indexes, and cascade-delete or bridge-delete nodes while preserving graph integrity.

n8n-cli node create <wf> --connect-from n0

Dry Run

Every mutating command supports --dry-run to preview changes without saving. Agents can validate their patches before committing, reducing errors to zero.

n8n-cli node update <wf> n1 --set ... --dry-run
Architecture

Built with precision

Clean separation between CLI, parser, and API client. The parser is the core — it's what makes node-level operations possible on top of a workflow-level API.

# Architecture
 
cmd/ # Cobra commands (thin: parse flags -> call service -> format)
workflow.go # CRUD, activate, deactivate, inspect
node.go # list, get, create, update, delete, rename, move
connection.go # list, create, delete edges
graph.go # topology analysis
 
internal/
client/ # n8n REST API client (Resty)
parser/ # THE CORE — fetch, parse, mutate, rehydrate, save
parser.go # Parse, Rehydrate, ResolveRef, ExtractPath
mutate.go # AddNode, UpdateNode, RemoveNode, AddEdge
graph.go # AnalyzeGraph (roots, leaves, topo sort, cycles)
service/ # Orchestration layer (client + parser)
output/ # Formatting (summary/details/json/params)
